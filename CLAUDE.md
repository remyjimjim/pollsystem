# CLAUDE.md

Guidance for Claude Code when working in this repository.

## Development log

This repo keeps a development log at `docs/DEVLOG.md`.

- **After every commit that changes the project, add a DEVLOG entry** in
  the *next* commit (an entry cites its change's commit hash, and a commit
  cannot contain its own hash).
- **Also log decisions reached in discussion that produced no code** —
  keeping a constraint, choosing not to build something, settling an
  approach — using a Decision entry.
- See the "Entry format" section at the top of `docs/DEVLOG.md` for the
  template, the entry variants (Changed / Verified / Decision), and the
  conventions. DEVLOG-entry commits are not themselves logged.
